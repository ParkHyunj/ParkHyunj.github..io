learning-redux / 2022.09.27 ~ 2022.09.29.

  • React를 배우는데, Redux가 자주 언급되어 노마더코더에서 Vanilla.js, React-Redux에 대하여 공부를 하게 되었습니다.
  • Redux는 기존의 프론트엔드 데이터 흐름을 관리하는 방식인 MVC 패턴(Model, View, Controller)의 단점을 없애고, 단방향 데이터 흐름의 특징 때문에 사용하게 되었습니다.
  • 업데이트 로직을 정의하는 함수인 리듀서를 사용함으로써, state 값(상태관리)을 컴포넌트 밖에서 관리할 수 있게 되었습니다.
  • Redux-toolkit를 설치하여 createAction, createReducer, configureStore, createSlice를 사용하면 action type, action creator, reducer 모두 작성을 하는데도, 기존의 Redux code보다 훨씬 간결해집니다.
  • 이 project를 통해 state를 관리할 수 있고, 변경 및 수정된 과정 또한 볼 수 있는 장점이 있는 Redux를 vanilla-redux, react-redux로 나눠 배웠습니다.

react-hooks / 2022.09.26. ~ 2022.09.27.

  • hook을 사용하면 컴포넌트로부터 상태 관련 로직을 추상화 할 수 있으며, 독립적인 재사용이 가능하며, class 없이 React 기능들을 사용할 수 있습니다.
  • useNotification는 버튼을 클릭하면, 설정한 알림이 실행되는 함수입니다.
  • usePreventLeave는 사용자들이 브라우저를 떠나기 전에 comfirm창을 띄어 확인을 받는 함수입니다. 주로 문서 작성 후, 저장하시겠습니까?와 같은 창을 만들 때 사용합니다.
  • useAxios는 axios를 다루는 hook으로, refetch()와 변수 trigger를 사용하여 버튼을 클릭하면, refetch()함수 실행 -> useState의 setTrigger 실행되어 새로운 데이터를 업데이트합니다.
  • 노마더코더에서 react-hooks에 대해 공부하면서, 13가지의 hook들에 대하여 습득하였습니다.
  • 현재는 package들의 버전이 많이 바뀌어서 사용하지 않거나 혹은 다른 hook으로 대체된 것들도 배웠습니다.
